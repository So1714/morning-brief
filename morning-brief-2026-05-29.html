GPT-5.5 ＆ Gemini 3.1 Ultra 相次ぎ投入——フロンティアAIがエージェント実行に軸足を移す

AI Updates Today / OpenAI / Google Cloud Blog  ｜  2026年4〜5月
OpenAIは2024年にGPT-4oをリリースし、その後GPT-5、GPT-5.5と急ピッチで世代交代を繰り返してきた。Googleも2025年のGemini 2.0シリーズからマルチモーダル性能を大幅向上させており、両社の競争は激化の一途をたどっている。

OpenAIは4月23日、GPT-5.5[1]をPlus・Pro・Business・Enterprise向けにロールアウトした。「最もスマートで直感的なモデル」と位置づけ、エージェント型コーディング・コンピューター操作・知識業務・科学研究に特化した設計を強調している。

Googleは同時期にGemini 3.1 Ultra[2]をリリース。動画・音声・テキストを転写工程なしに同時処理する「真のマルチモーダル」を初めて商用規模で実現した。コード実行サンドボックスが標準装備され、モデルが会話中に自律的にコードを書き・実行・テストするワークフローが可能になった。軽量版のGemini 3.1 Flash-Liteは200万トークンコンテキストを$0.25/Mトークンで提供する。

金融システム開発への示唆は大きい。照合処理・取引監視・契約審査といったルーティンワークの自動化精度が跳ね上がり、AIがシステムを直接操作するコンピューターユース[3]機能は勘定系周辺のRPA代替に使える可能性を秘める。ただし機密データ処理における自社ホスティング[4]対応の検討が急務となる。

注釈を開く
  1. GPT-5.5：OpenAIが開発した大規模言語モデルの第5.5世代。推論・コーディング・マルチモーダル処理能力を高め、エージェント型タスクに最適化。
  2. Gemini 3.1 Ultra：Googleのフロンティア言語モデル。テキスト・画像・音声・動画を統合処理するマルチモーダル能力が特徴。
  3. コンピューターユース（Computer Use）：AIが人間のようにGUIを操作してWebブラウジング・アプリ操作などを行う機能。Anthropicが2024年に一般公開し各社が追随している。
  4. 自社ホスティング（Self-hosting）：クラウドAPIを使わず、自社インフラ上でモデルを稼働させること。外部へのデータ送出を避けたい金融機関で特に重要。

GitHub Trending / OSSInsight / AIToolly  ｜  2026年5月
2024年末頃からAnthropicが公開したMCP（Model Context Protocol）標準が普及し始め、AIエージェントがシステムを横断して情報取得・操作できる「マルチエージェント」アーキテクチャが2025〜2026年の開発トレンドになっている。

金融AIエージェントではTradingAgents[1]が注目を集めている。ファンダメンタル分析・センチメント分析・テクニカル分析・トレーダー・リスクマネージャーなど役割を分担した複数エージェントが連携し、実際の取引ファームの動態を模倣する。

開発ツールではDSPy[2]（プロンプトを機械学習で最適化）、LangGraph[3]（ステートフルなグラフ型エージェントワークフロー）、Browser Use[4]（AI向けWebブラウザ操作）が実務採用を広げている。金融システム開発者にとっては、レガシーAPIラッパー生成・ドキュメント検索・テスト生成での活用が現実的な入口となっている。

注釈を開く

  2. DSPy：スタンフォード大学発のフレームワーク。手動プロンプトエンジニアリングの代わりに機械学習でプロンプトを最適化する。

  4. Browser Use：AIエージェントがWebブラウザを操作するためのライブラリ。人間がブラウザでできる操作を自動化する。

ISO 20022、2026年11月が「本当のデッドライン」——非構造化アドレスは全面拒否へ

SWIFT公式 / Kyriba / The Paypers / Finextra  ｜  2026年5月
SWIFTのISO 20022移行計画ではMT/MX共存（コエグジステンス）期が2025年11月に終了した。日本の主要行を含む多くの金融機関がインフロー翻訳に依存したまま事実上の延命を続けてきた経緯がある。

2026年11月14日から、SWIFTは照会・調査（E&I）メッセージをMX形式（camt.110 / camt.111[1]）での受信を全金融機関に義務付ける。同日付で非構造化アドレス[2]を含む支払いが自動的に拒否されるようになり、構造化国コード・都市名の必須化が実施される。

2026年1月からはコンティンジェンシー処理[3]およびインフロー翻訳サービス[4]への追加課金がすでに始まっており、移行先延ばしには財務コストが発生している。Bank of Americaの決済プロダクト責任者は「ISO 20022は今日の決済業界で最も難しいトピック」と述べており、エンドツーエンドでの構造化データ保持が業界課題のままであることを示している。

国内の金融システム開発チームは、コルレス銀行[5]チェーン全体での構造化データ維持に向けたシステム改修の優先度を今一度確認すべき時期にある。

注釈を開く
  1. camt.110 / camt.111：ISO 20022のMX（XML形式）メッセージタイプ。camt（Cash Management）系は口座照会・通知・調査に使用。旧MT形式のMT110等に相当。
  2. 非構造化アドレス：受取人住所を自由テキストで入力したもの。ISO 20022では国コード・都市名・番地を別フィールドに分割した「構造化アドレス」が必須となる。
  3. コンティンジェンシー処理（Contingency Processing）：MX形式送信ができない金融機関が緊急的に旧MT形式を使い続けるための暫定サービス。SWIFTが追加手数料を課している。
  4. インフロー翻訳サービス（In-flow Translation Service）：受信したMT形式メッセージを自動的にMX形式へ変換するSWIFTのサービス。受信行が未移行の場合の救済措置だが、2026年1月から有料化。
  5. コルレス銀行（Correspondent Bank）：国際送金において送金銀行と受取銀行の間を仲介する銀行。クロスボーダー送金は通常、複数のコルレス銀行を経由する。

リアルタイム決済×ISO 20022 APIがB2Bトレジャリーを変革——2026年が「真の変曲点」

Finextra / The Paypers / FRB Services  ｜  2026年5月
米国のFedNow（2023年稼働）、英国のFaster Payments、欧州のSEPA Instantなどリアルタイム決済（RTP）インフラが各地に整備されてきた。しかし企業財務（トレジャリー）への本格適用は遅れており、24/7の即時確認・リアルタイム不正対応が課題となってきた。

ISO 20022のエンドツーエンド構造化データとオープンバンキングAPIが組み合わさることで、異なるRTPプラットフォーム間の相互運用性が現実的になりつつある。専門家は「2026年が実際の変曲点」と位置づけ、自動照合・即時流動性把握・リアルタイム不正検知の三機能が同時実現するシナリオを描く。

金融システム開発者の観点では、バッチ処理[1]からリアルタイム処理へのパラダイムシフトへの対応としてイベント駆動アーキテクチャ[2]への移行とレイテンシ要件の再定義が開発優先課題として浮上している。

注釈を開く
  1. バッチ処理：一定期間（例：1日）に溜まったトランザクションを一括処理する方式。夜間バッチが典型例。リアルタイム処理の対義語。
  2. イベント駆動アーキテクチャ（Event-Driven Architecture）：取引発生などの「イベント」を即座に検知し、それをトリガーとして処理を連鎖させるシステム設計。KafkaなどのメッセージブローカーがI基盤を担う。

Mastercard、ステーブルコインインフラのBVNKを18億ドルで買収——B2Bクロスボーダー決済に本格参入

Mastercard IR / CNBC / Forrester / Fortune  ｜  2026年3月17日発表
Mastercardは従来からトークン化資産への関心を示してきたが、2025年にステーブルコインのB2B決済での存在感が急拡大。B2Bステーブルコイン取引は前年比733%増加し、ステーブルコイン決済全体の約60%を占めるに至った。

Mastercardは3月17日、ロンドン拠点のBVNK[1]を最大18億ドル（うち3億ドルはパフォーマンス連動型）で買収することで合意した。BVNKは130カ国以上でステーブルコイン取引を可能にするインフラを保有する。

戦略的狙いは「オンチェーン決済とフィアット[2]レールの接続」。B2B送金・クロスボーダー送金・リミッタンスにおいて、ステーブルコインをMastercardのグローバルfiatネットワーク上で法定決済手段として扱えるようにする。Forresterはこれを「B2Cカード処理の構造的陳腐化へのヘッジ」と分析している。

買収完了は2026年後半の予定。BVNK統合後はMastercardのトークン化資産プラットフォーム[3]に組み込まれる見通し。日本の銀行にとっては、MastercardとのコルレスTBサービスが中長期的に変質するリスク・機会として継続注視が必要。

注釈を開く
  1. BVNK：2021年創業のロンドン拠点フィンテック。企業向けステーブルコイン決済インフラを提供し、130カ国以上でUSDC・USDTなどのステーブルコイン取引を支援。
  2. フィアット（Fiat）：法定通貨のこと。円・ドル・ユーロなど各国政府が発行する通貨。ステーブルコイン（デジタル資産）と対比して使われる。
  3. トークン化資産プラットフォーム（Mastercard Tokenized Asset Platform）：Mastercardが金融機関向けに提供するプラットフォーム。自社ステーブルコインやトークン化預金の発行・管理機能を提供する。

Visa、ステーブルコイン決済を本格展開——カード紐付け・法人ペイアウト・独自発行プラットフォームの三本柱

FinXTech / AlphaPoint  ｜  2026年5月
Visaは2023〜2024年にかけてステーブルコインの決済インフラ接続を段階的に実証してきた。2025年以降は法人・B2B用途への応用が急速に広がっている。
