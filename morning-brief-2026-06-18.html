GitHub Copilot、6月1日より従量課金制に完全移行 ― 1クレジット＝0.01ドル、AIエージェント利用の費用が組織の新課題に
ソース：GitHub Blog / GitHub Changelog  |  2026年6月1日
GitHub Copilotは2021年の提供開始以来、フラット月額制でコード補完機能を提供してきた。2023年以降、チャットやエージェント機能が急拡大し、モデルごとのトークン消費量の差が収益構造に不整合を生じさせ始めた。2025年秋にMicrosoftはトークン課金への移行を予告し、2026年6月1日をもって全プランを正式移行した。

GitHub Copilotの全プランが2026年6月1日付で従量課金制（AIクレジット制）へ移行した。月額サブスクリプション料がそのままクレジット残高に変換される仕組みで、1クレジット＝0.01ドル。Copilot Pro（月10ドル）は10ドル分のクレジット、Enterprise（ユーザーあたり39ドル）は同額分のクレジットを毎月付与する。

コード補完・Next Edit Suggestions*1はクレジット消費対象外となり引き続き無制限で利用できるため、補完のみ活用するチームへの実質的影響は軽微。一方、Claude Opus 4.8やGPT-4oを呼び出すエージェントモード*2での大規模コードレビューや多ファイル編集は、トークン消費が多く月額上限を超えるリスクがある。

組織・Enterprise向けにはユーザー単位の予算上限設定が可能となり、AIコスト管理の新たなコントロールポイントが生まれた。金融システム開発など大規模プロジェクトでエージェントを常用するチームは、従来の「使い放題」の前提を見直す必要がある。

注釈
  1. Next Edit Suggestions（NES）：コード編集中に次に変更すべき箇所を自動予測し提案する機能。Copilotが過去の変更パターンを解析して先読みする。
  2. エージェントモード：AIが自律的に複数のツール呼び出し・ファイル操作・テスト実行を連続して行い、より複雑なタスクを遂行するモード。単純な補完よりもトークン消費量が大幅に多い。

ソース：TechCrunch / Axios  |  2026年6月4日
Rampは2019年創業のコーポレートカード・経費管理プラットフォームで、2022年にはユニコーン入りした。2025年7月時点の評価額は約160億ドルだったが、AIを活用したコスト削減機能の訴求が投資家に評価され、同年11月に220億ドル、2026年3月に320億ドルと急騰していた。

Rampは6月4日、ICONIQ・GIC・オンタリオ州教職員年金（OTPP）主導のSeries Fで7.5億ドルを調達し、評価額が440億ドルに到達したと発表した。新たな投資家にはゴールドマン・サックス・オルタナティブス、D.E. Shaw、モルガン・スタンレーIM等が名を連ねる。

現時点の年換算収益（ARR）は15億ドル超（同社は昨年9月に10億ドル突破を発表済み）で、顧客数は7万社超。Visa・Uber・Shopify・Andurilなども顧客に含まれ、年間決済処理額は2000億ドルを超える。

調達資金の主要用途は、AIエージェントによる調達*1・会計・支出インテリジェンスの自動化と、国際展開。同社は従来の経費管理SaaS*2から「AIが稟議・承認・照合を自律実行するFinOps*3プラットフォーム」へのポジション転換を明確にしている。金融システム開発者にとっては、ERP*4連携APIの設計標準が変わるシグナルとして注目に値する。

注釈
  1. 調達（Procurement）：企業が外部から物品・サービスを購入する一連の業務プロセス。発注・承認・受領・支払照合を含む。
  2. SaaS（Software as a Service）：クラウド経由でソフトウェアをサブスクリプション提供するモデル。
  3. FinOps（Financial Operations）：クラウドや企業支出コストを可視化・最適化するプラクティス。DevOps的な反復サイクルを財務管理に適用する。
  4. ERP（Enterprise Resource Planning）：会計・購買・在庫・人事などを統合管理する基幹システム。SAPやOracleが代表的。

ソース：The Block / GlobeNewswire  |  2026年6月10日
Figure Technologyは、HELOCをブロックチェーン上でトークン化する手法で2018年に創業。RWA（現実資産）トークン化市場で75%のシェアを持つとされ、2025年以降に機関投資家向け住宅ローン市場のデジタル化を加速させてきた。Kiaviは不動産投資家向けの短期リハブローンと長期賃貸ローンを専門とするAIレンダー。

Figure Technology Solutionsは6月10日、AIレンダーKiaviを7.17億ドルで買収する最終合意に至ったと発表した。取引スキームは、Sixth Streetが組成するJVがKiaviのバランスシート資産（貸付残高）を購入し、Figureがテクノロジー・運営プラットフォームを取得するという二段構え。

Kiaviは月間1億ドル超の住宅ローン組成フロー*1を有しており、買収完了後はFigureのブロックチェーンネイティブなウェアハウスマーケットプレイス「Democratized Prime」*2に組み入れられる予定。これにより年間70億ドル規模の新規ローン資産がオンチェーン*3化される見込みだ。

FigureはAI製品「Adaptor」も同時に公表。エージェント間（A2A）のオンボーディング*4を担い、異なるデータスキーマを持つオリジネーター*5からのデータを統一フォーマットに変換する。RWA市場において「住宅ローンの証券化フロー自体をオンチェーン完結させる」というビジョンが具体化しつつあり、銀行のカストディ*6・決済システムへの影響を見極める必要がある。

注釈
  1. ローン組成フロー（Loan Origination Flow）：新規ローン契約の月間実行額。
  2. Democratized Prime：Figureが運営するブロックチェーン上の機関投資家向け住宅ローン市場プレイス。ウェアハウスレンディング（銀行の融資枠でローンを一時的に保有）をオンチェーンで行う。
  3. オンチェーン（On-chain）：ブロックチェーンネットワーク上に記録・処理されること。
  4. A2Aオンボーディング（Agent-to-Agent Onboarding）：AIエージェント同士が自律的にデータ交換・検証を行いながら新規業者を登録するプロセス。
  5. オリジネーター（Originator）：ローンを実際に組成・実行する金融機関または貸付業者。
  6. カストディ（Custody）：金融資産の保管・管理業務。デジタル資産の文脈では秘密鍵の安全な保管も含む。

ソース：llm-stats.com / DEV Community / Axios  |  2026年6月
2023年3月のCognition AI「Devin」発表を契機に、AIによる自律的コーディングエージェントの開発競争が始まった。2024年はGitHub Copilot Agentモード、Cursor、Windsurfなどのツールが台頭し、2025年にはAnthropic「Claude」、OpenAI「GPT-4o」が大幅な性能向上を達成。2026年は商用・オープンソースともに「本番コードを書ける」水準に到達しつつある。

2026年6月時点のベンチマーク評価（SWE-bench Verified*1）では、OpenAIのGPT-5.5が長文エージェントタスク全般でトップを維持する一方、AnthropicのClaude Opus 4.8（2026年5月28日リリース、入力5ドル/Mトークン・出力25ドル/M）がコード生成タスクのSWE-benchで首位を取るケースも報告されている。

OSSエージェントフレームワークでは、OpenHandsがSeries A（1880万ドル）調達後にGitHub Stars 7万超を達成。AMD・Apple・Google・Amazon・Netflix・NVIDIAのエンジニアが貢献するコミュニティに成長した。新興プラットフォームでは「agnt8x」（EightX Labs）が6月8日にパブリックプラットフォームを公開し、マルチエージェントオーケストレーション*2向けの統一ガバナンス機能を提供し始めた。

金融システム開発の文脈では、コード補完から「複数システムを跨いだ結合テスト*3の自律実行」へとエージェントの射程が広がりつつある。セキュリティ審査・コンプライアンス*4チェックをパイプラインに組み込む設計標準の整備が急務となっている。

注釈
  1. SWE-bench Verified：実際のGitHubイシューを解決する能力を測るAIコーディングエージェントの標準ベンチマーク。人手で検証済みのテストセットを使用。
  2. マルチエージェントオーケストレーション：複数のAIエージェントが役割を分担しながら協調してタスクを遂行する仕組み。指揮役の「コンダクター」エージェントが全体を制御する。
  3. 結合テスト（Integration Test）：複数のシステムコンポーネントを組み合わせた際の動作を確認するテスト工程。
  4. コンプライアンス（Compliance）：法令・規制・社内ルールへの準拠。金融分野ではFISC安全対策基準やPCI DSS等が該当。

Mastercard、ステーブルコイン決済を8チェーン・6通貨に拡張 ― 「24時間365日決済」へ向けバックオフィスが変わる
ソース：Mastercard公式 / CoinDesk / Ledger Insights  |  2026年6月3日
Mastercardは2021年頃からステーブルコインの決済インフラへの組み込みを段階的に検討し、2023年にはCircleとのパイロットを実施。2025年には複数チェーンへの対応を予告していた。伝統的なカード決済では、銀行の営業時間外（週末・祝日・夜間）は最終決済が翌営業日以降に持ち越されるため、流動性コストが発生する構造的課題があった。

Mastercardは6月3日、カード決済のオンチェーン決済機能を大幅拡張すると発表した。対応ステーブルコインは、CircleのUSDC、Ripple発行のRLUSD*1、PayPalのPYUSD、PaxosのUSDGおよびUSDPを含む計6通貨。ブロックチェーンネットワークはEthereum・Solana・Polygon・Base・Arbitrum・XRPLの8チェーン（Baseは同一チェーンファミリーを含む）に対応する。

重要な点は、これが消費者向けの「暗号通貨払い」ではなく、イシュアー*2とアクワイアラー*3間の「バックオフィス決済」の変革である点だ。消費者は従来通り法定通貨建てで支払い、ネットワーク内部の銀行間資金移動部分にステーブルコインレールが使われる形となる。

初期パートナーにはARQ（旧DolarApp）、Cross River Bank、Lead Bank、Nuveiが名を連ねる。この変更により、週末・祝日の日中決済が可能となり、銀行のイントラデイ*4流動性管理に直接影響する。トランザクションバンキング部門にとっては、ネットセトルメント*5サイクルの見直しが迫られる可能性がある。

注釈
  1. RLUSD：Ripple社が発行するドルペッグのステーブルコイン。XRP Ledgerおよびイーサリアム上で発行される。
  2. イシュアー（Issuer）：クレジットカードやデビットカードを発行する金融機関。カード会員との直接契約関係を持つ。
  3. アクワイアラー（Acquirer）：加盟店（店舗側）の決済を処理・代行する金融機関。
  4. イントラデイ流動性（Intraday Liquidity）：営業時間内に随時発生する資金繰りニーズ。決済システムの資金ポジション管理において中心的な概念。
  5. ネットセトルメント（Net Settlement）：一定期間の取引を相殺・集約してから最終決済する方式。グロス決済（1件ずつ即時決済）と対比される。
JPMorgan・Citi等12行、The Clearing House主導のトークン化デポジットネットワーク計画を発表 ― ステーブルコインへの防衛戦略として2027年上半期を目標
ソース：CoinDesk / Unchained Crypto / American Banker  |  2026年6月5日
米国では2025年の「GENIUS Act」等の立法整備によりステーブルコインの法的地位が明確化され、Tether・Circleが発行するUSDTとUSDCの流通量がさらに拡大。銀行のデポジット（預金）がステーブルコインへ流出するリスクが顕在化してきた。The Clearing House（TCH）は既存のACHやRTPを運営する米国の主要決済インフラ運営体で、加盟銀行の共有機関として機能する。

JPMorgan Chase、Citigroup、Bank of America、Wells Fargo、BNY、HSBC、PNC、US Bank、Truist、Santander、TD Bank、Regionsの12行は6月5日、TCH（The Clearing House）が運営するトークン化デポジットネットワーク*1の構築計画を正式に公表した。目標時期はWSJが報じた2027年上半期。

このネットワークでは、顧客の銀行預金をブロックチェーンベースのトークンに変換し、24時間365日を問わず参加行間で即時移転できるようにする。ステーブルコインと異なり、トークンは発行銀行のバランスシート上に残り、FDIC*2預金保険も引き続き適用される規制内の仕組みだ。

ただし、6月時点では参加ブロックチェーンベンダーの選定・ルールブック策定・ネーミング・料金体系はいずれも未決定で、公式なローンチ日の発表もない。日本のメガバンクの観点では、コルレス銀行*3ネットワークやSWIFT経由の米ドル決済フローへの影響を注視すべき局面に入ってきた。

注釈
  1. トークン化デポジット（Tokenized Deposit）：銀行預金をブロックチェーン上のトークンとして表現したもの。ステーブルコインと異なり、既存の銀行規制の枠内で発行・管理される。
  2. FDIC（Federal Deposit Insurance Corporation）：米国連邦預金保険公社。1口座あたり最大25万ドルを保証。
  3. コルレス銀行（Correspondent Bank）：海外送金や外貨決済において、送金元・受取先の銀行に代わって決済を仲介する銀行。ドル決済ではJPMorganやCitiがグローバルコルレス銀行として機能することが多い。

インド最大規模の法人銀行イベント「CBNxT'26」開幕 ― Standard Chartered・HSBC・DBSがAI・キャッシュマネジメント刷新を議論
ソース：Business Standard / Outlook Business  |  2026年6月4日
インドは2022年以降、UPI（統一決済インターフェース）の急普及とRBI（インド準備銀行）主導の金融デジタル化政策により、法人向けキャッシュマネジメントの変革が他市場に先行して進行している。外資系銀行（Standard Chartered・HSBC・DBS）は、インドの大企業・SME向けトランザクションバンキングを成長ドライバーとして位置付け、現地投資を積極化させている。

「CBNxT'26」は6月4日にムンバイのSofitel BKCで開幕した、インド初の法人向けキャッシュマネジメント・TB専門カンファレンス。Standard Chartered・HSBC・DBS・Yes Bank・HDFC Bankなどから経営幹部が参集した。

主要討議テーマはAIを活用したキャッシュポジション予測*1、組込型金融（Embedded Finance）*2、API*3ファーストなバンキングシステム、クロスボーダー決済のリアルタイム化、トレードファイナンス*4の電子化。DBSはインド中小企業（SME）向けバンキングのヘッドが登壇し、API統合型ソリューションのロードマップを示した。

本イベントは直接的なプロダクト発表の場ではないが、外資TB行がインド市場で何を優先課題とするかを示す指標として注目される。

注釈
  1. キャッシュポジション予測：将来の入出金を予測して資金余剰・不足を事前に把握する業務。AIにより精度向上と自動化が進んでいる。
  2. 組込型金融（Embedded Finance）：非金融系のアプリやサービスに金融機能（決済・融資・保険等）を埋め込む形態。ERPや物流システムへの直接統合が典型例。
  3. API（Application Programming Interface）：システム間でデータや機能を安全に受け渡すための接続仕様。オープンバンキングの基盤技術。
  4. トレードファイナンス（Trade Finance）：輸出入取引の代金決済を支援する金融サービス群。信用状（LC）・荷為替手形・ファクタリングなどを含む。

高市政権、外国人による不動産取得規制を先送り ― 改正法案の優先度をめぐり与党内で調整続く
ソース：Nippon.com / 首相官邸  |  2026年6月
