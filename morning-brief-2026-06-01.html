MCP×金融：NSAがセキュリティ警告を発令、Fiservが「agentOS」を商用化――採用が安全対策を追い越す

ResultSense / Fiserv IR · 2026年5月26日
MCP（Model Context Protocol）はAnthropicが2024年末にオープンソース公開したプロトコルで、AIアシスタントが外部システム・データに安全にアクセスするための標準仕様。2025年前半より金融機関への試験導入が始まり、2026年4月時点で世界の銀行・保険・証券における実証事例は200件超に拡大した。

米国家安全保障局1（NSA）は5月26日、金融機関のMCP2導入を念頭に置いたセキュリティガイダンスを公表し、認証の脆弱性・不十分な承認制御・監査ログの欠如・インストラクション・インジェクション3リスクを主要な課題として列挙した。金融機関向けアジェンティックAI4の採用は2026年に前年比600%超の拡大が見込まれており、急速な採用がセキュリティ対応を追い越しつつある実態が浮き彫りとなった。

一方でFiserv5は同月、「agentOS」を正式ローンチした。KYC自動化・不正検知・コンプライアンスレポート生成を単一プラットフォームで実行するバンキング向けエージェント基盤だ。2026年4月にはMeow Technologiesが「世界初のエージェンティック・バンキング・プラットフォーム」を開始し、AIエージェントによる法人口座開設・バーチャルカード発行・送金・請求書管理の自動化を実現した。

【分析】 MCPは金融システム開発に直結する技術仕様であり、社内システムへのAIエージェント接続をAPI6経由で標準化する点が重要。NSAの警告は「採用先行・セキュリティ後付け」への警鐘であり、国内金融機関がMCP対応を進める際は認証・承認フローの設計段階からセキュリティ要件を組み込む必要がある。44%の金融機関が2026年中にアジェンティックAIを採用する見通しを踏まえると、MCPセキュリティガイドラインは今後12カ月で事実上の業界標準要件となる可能性が高い。

▶ 注釈を展開（6項目）
  1. NSA（National Security Agency）：米国家安全保障局。通信・電子情報の収集・分析を担う米情報機関で、サイバーセキュリティ指針の発行も行う。
  2. MCP（Model Context Protocol）：AnthropicがオープンソースとしてリリースしたAIとツール群を繋ぐ標準プロトコル。銀行API・データベース・社内システムへのAI接続を標準化し、開発コストを削減する。
  3. インストラクション・インジェクション（Instruction Injection）：外部から悪意ある指示をAIに注入し、意図しない動作を引き起こす攻撃手法。SQLインジェクションのAI版に相当する。
  4. アジェンティックAI（Agentic AI）：単に質問に回答するだけでなく、タスクを計画し、ツールを使い、複数ステップにわたって自律的に実行するAIシステム。
  5. Fiserv：米国を代表するFinTechインフラ企業。銀行・信用組合向けにコアバンキング・ペイメント処理を提供。世界1万社超の金融機関を顧客に持つ。
  6. API（Application Programming Interface）：ソフトウェア同士が機能・データを呼び出し合うためのインターフェース仕様。銀行システム間の連携でも広く使用される。

Visa傘下Pismoが警告：AIがレガシー問題を露呈――「モダナイゼーションは技術交換でなく戦略判断」

PYMNTS.com · 2026年5月
Visaは2023年6月にブラジル発のクラウドネイティブ・コアバンキングプラットフォームPismoを約10億ドルで買収。以降、VisaのイシュアープロセッシングをPismoのAPI-firstアーキテクチャで刷新するプロジェクトが進行中。2025年には主要欧州行を含む複数のPismoクライアントがAIデプロイでの制約を報告し始めた。

Visa傘下のPismo1は、AI導入がレガシーコアシステム2の限界を可視化していると指摘した。多くの銀行では新たなAI機能をローンチしたくても、テクノロジー変更キューが数年単位で積み上がっており、製品改善を時間軸で実現できない状況が続く。Pismo幹部は「モダナイゼーションは技術のアップグレードでなく戦略的な選択」と述べ、モノリシックな既存システムを一括置換するのではなく、モジュール型・API-first3なアーキテクチャを既存基盤と並走させる段階的アプローチを推奨した。

Visaの関連レポートでは、Pismoのクラウドネイティブ4プラットフォームを採用した銀行が新プロダクトのローンチ時間を従来比70〜80%短縮した事例が示された。レガシー環境で頻発するデータサイロ5問題がAI統合の最大の障壁となっていることも改めて確認された。

【分析】 国内メガバンクも勘定系の次世代移行を検討する局面にあり、「AIをいかに素早くオンボードできるか」がシステム設計の評価軸として浮上しつつある。Pismoモデルが示すのは、全面刷新が困難な場合でもAPI層を標準化することで段階的に機能強化できるというパスであり、国内SIer・ベンダーとの交渉でも参照できる論理軸となる。

▶ 注釈を展開（5項目）
  1. Pismo：2016年ブラジル創業。クラウドネイティブなコアバンキング・カードイシュアリングプラットフォームを提供。2023年にVisaが約10億ドルで買収。
  2. レガシーコアシステム（Legacy Core System）：数十年前に設計された銀行の基幹システム。COBOLなど旧来技術で動作し、変更に時間と費用がかかる。
  3. API-first：全機能をAPIとして設計・公開することを前提としたアーキテクチャ思想。外部システムとの連携・機能の再利用が容易になる。
  4. クラウドネイティブ（Cloud-native）：クラウド環境の特性を活かして設計されたシステム。コンテナ化・マイクロサービス・CI/CDが特徴でスケーラビリティと変更速度が高い。
  5. データサイロ（Data Silo）：組織・システム内でデータが孤立し、他システムと共有・連携されていない状態。AIが必要とするデータ統合の妨げとなる。

▶ 注釈を展開（5項目）

SWIFTのブロックチェーン共有台帳、2026年内に実取引で稼働へ――EVM互換・Hyperledger Besuベースで構築

Finextra · 2026年
SWIFTは2025年9月のSibosで共有台帳の構築計画を正式発表し、グローバル銀行コンソーシアムとの設計フェーズを経て2026年のMVP稼働を宣言。2023〜2025年にかけてCBDC・トークン化資産の相互運用性PoCを複数完了しており、BNPパリバとのトークン化資産取引オーケストレーションPoCも成功している。

SWIFTは2026年内に、ブロックチェーン1ベースの共有台帳2のMVP3を実際のトランザクションで稼働させると発表した。技術基盤はEVM互換4のHyperledger Besu5（オープンソース）で、SWIFTが台帳の運用主体となりインターバンク決済コミットメントの記録・検証・ワークフロー調整を担う。台帳はトークン化預金6を価値の表現形式として活用し、既存のSWIFT標準との互換性を維持しながら複数の決済オプションをサポートする。

【分析】 SWIFTが自らブロックチェーン台帳を運用する構造は、同ネットワークの「プラットフォーム化」を示す。メガバンクのTB部門にとっては、既存のSWIFT接続を変更せずにトークン化決済インフラに乗れる可能性があり、移行コストが低い「ハイブリッド移行」の現実的な選択肢となる。2026年末のMVP稼働状況は来年の予算・システム計画に直結する注目ポイントだ。

▶ 注釈を展開（6項目）
  1. ブロックチェーン（Blockchain）：取引データを暗号技術でチェーン状に連結した分散型台帳技術。改ざんが困難でトランザクションの透明性が高い。
  2. 共有台帳（Shared Ledger）：複数の参加者が共同で読み書きできる分散型のデータ記録基盤。銀行間の資金移動・資産取引の記録に活用される。
  3. MVP（Minimum Viable Product）：最小限の機能で市場に投入する初期バージョン。本番環境での早期フィードバック取得が目的。
  4. EVM互換（Ethereum Virtual Machine Compatible）：Ethereumのスマートコントラクト実行環境と互換性を持つこと。Solidityで書かれたコードが流用でき、エコシステムの活用が容易になる。
  5. Hyperledger Besu：Linux Foundationが管理するエンタープライズ向けEVM互換ブロックチェーン。プライベート・コンソーシアムチェーンの構築に適しており、金融機関での採用実績が増えている。
  6. トークン化預金（Tokenized Deposit）：銀行預金をブロックチェーン上のデジタルトークンとして表現したもの。CBDCとは異なり発行主体は民間銀行。決済の即時性・プログラマビリティが高まる。

JPMorganとCitiは4月、双方のグローバル決済責任者（Umar Farooq・Shahmir Khaliq）がBloombergの共同インタビューに登場し、ブロックチェーン決済の覇権を巡る競争を公言した。JPMorganのKinexys1プラットフォームは現在、デポジットトークン2（JPM Coin）を用いた機関間決済で日次50億ドル超を処理し、24時間365日のリアルタイム決済を実現。一方CitiはCoinbase3との提携を拡大しCiti Token Services4を通じてトレードファイナンス・クロスボーダー送金のブロックチェーン化を進める。両行はトークン化決済レール5の獲得を次世代グローバル決済インフラの中核と位置付け、SWIFTや新興ステーブルコイン勢との競合環境の中でエンタープライズ顧客への囲い込みを狙う。

▶ 注釈を展開（5項目）
  1. Kinexys：JPMorganのブロックチェーンベース決済・デジタル資産プラットフォーム。旧称「Onyx」から2023年に改称。JPM Coinを用いた機関間リアルタイム決済が中核サービス。
  2. デポジットトークン（Deposit Token）：銀行が自行の預金をブロックチェーン上のトークンとして表現したもの。JPM Coinはその代表例。既存の預金保護の枠組みに収まる点がステーブルコインと異なる。
  3. Coinbase：米国最大の暗号資産取引所。機関投資家向けの暗号資産カストディ・決済サービスも展開し、大手銀行との提携を拡大している。
  4. Citi Token Services：Citiがトレードファイナンス・クロスボーダー決済のトークン化に特化して提供するサービスブランド。スマートコントラクトを用いた条件付き決済自動化が特徴。
  5. トークン化決済レール（Tokenized Payment Rail）：ブロックチェーン上でトークン化された価値を移転するための決済インフラ。SWIFT/ACHなどの法定通貨レールに代わる次世代インフラとして注目される。
